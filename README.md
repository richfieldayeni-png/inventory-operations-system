# Inventory & Operations Tracking System

## Overview
A backend-focused inventory and operations tracking system designed for
distribution and logistics environments.

The system models real-world depot operations by tracking products,
inventory levels, deliveries, driver activity, and operational expenses.

It is built to provide management with structured, data-driven visibility
into daily operations.

---

## Core Features
- Product and inventory management
- Stock movement tracking (inbound and outbound)
- Driver and delivery logging
- Expense tracking linked to operational activity
- Operational reporting for decision-making

---

## Key Report Example
### Daily Delivery Performance & Cost Report
Provides a per-driver summary of:
- Number of deliveries completed
- Total units delivered
- Associated operational expenses

This report helps management evaluate delivery efficiency and cost
performance on a daily basis.

---

## System Design
- Relational database schema with normalized tables
- Python-based backend logic for operational workflows
- SQL aggregation queries for business reporting

---

## Tech Stack
- Python
- MySQL
- Git

---

## Use Case
Designed for small-to-medium distribution operations that require
accurate inventory tracking and daily operational reporting but lack
dedicated enterprise systems.
